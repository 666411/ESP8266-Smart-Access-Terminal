 🔨 硬件抗干扰物理重构 (Hardcore Debug)
在原型研发期，遭遇了极其隐蔽的“继电器动作导致单片机频繁异常复位”的物理级 Bug。
经过示波器与万用表的联合排查，精准定界为：大电流感性负载（电磁锁线圈）在断电/吸合瞬间产生的巨大反向电动势（Back EMF），击穿了核心板的 3.3V 供电网络。

最终解决方案： 摒弃无效的软件延时看门狗思路，从物理底层重构电路。引入高速光电耦合器彻底切断控制端电气连接，并设计了双电源独立供电系统（仅信号共地），彻底拔除电磁干扰隐患，保障了边缘节点 100% 的运行鲁棒性。

 ⚙️ 嵌入式软件架构亮点

   干掉 delay()，非阻塞状态机觉醒：** 摒弃传统的死循环阻塞思维。利用 `millis()` 系统滴答时钟重构主循环，实现了扫码中断读取、OLED 动态刷新与 HTTP 远程指令查询的“伪并发”处理。确保高延迟的网络请求绝对不阻塞底层的传感器响应。
   强韧的断网容错机制：** 面对恶劣的工业 Wi-Fi 环境，开发了自带指数退避算法的断线重连守护机制，防止网络模块假死。
   基于 C++ 的模块化抽象：** 熟练运用 PlatformIO 及 Arduino 框架，分别驱动了 SSD1306 (I2C) 及 MH-ET 扫码模块 (SoftwareSerial)，并通过严格的数据帧校验防御串口粘包。
